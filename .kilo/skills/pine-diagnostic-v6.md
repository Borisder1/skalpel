# Pine Script v6 - Навичка діагностики торгівельних стратегій

## Огляд
Ця навичка забезпечує комплексну діагностику торгівельних стратегій Pine Script v6, виявляючи помилки, логічні невідповідності, проблеми продуктивності та можливості оптимізації.

## Основні області діагностики

### 1. Аналіз потоку виконання стратегії
```pinescript
// ПЕРЕВІРКИ ДЛЯ ВИКОНАННЯ:
// - Перевірка переходів AgentPhase (Perception → Reasoning → Planning → Execution)
// - Валідація цілісності стану (Idle → Armed → InPosition → Cooldown)
// - Перевірка timing та qty_percent для strategy.entry/exit
```

### 2. Рамкова виявлення критичних помилок
```pinescript
// КРИТИЧНІ ПОМИЛКИ ДЛЯ ВИЯВЛЕННЯ:
// - Гонка замовлень стратегії
// - Поширення NaN у розрахунках ризику
// - Вихід за межі масиву при виявленні патернів
// - Неспрямованість lookahead у функції Security()
// - Перебірка pivot із look-ahead bias
```

### 3. Аналіз вузьких місць продуктивності
```pinescript
// ПОКАЗНИКИ ПРОДУКТИВНОСТІ:
// - Кількість ліній/міток/коробок у межах лімітів
// - Оптимізація кількості security викликів
// - Зростання масивів без перевірки меж
// - Переписування патернів при confirmed vs every-tick logic
```

## Процедура діагностики

### Фаза 1: Аудит синтаксису та компіляції
1. Перевірка відповідності `//@version=6`
2. Валідація всіх enum оголошень та використань
3. Перевірка підписів методів UDT (User Defined Type)
4. Підтвердження max_bars_back оголошень

### Фаза 2: Валідація логічного потоку
1. Відстеження створення → анулювання order block
2. Перевірка таймінгу виявлення liquidity sweep
3. Перевірка стану imbalance sequence
4. Валідація виявлення патернів без look-ahead

### Фаза 3: Перегляд управління ризиком
1. Точність розрахунку розміру позиції
2. Перевірка SL/TP рівнів
3. Забезпечення cooldown механізму
4. Обмеження сигналів за день

### Фаза 4: Аналіз торгівельної ефективності
1. Точність розрахунку відсотка виграшів
2. Статистична валідність context scoring
3. Математична коректність similarity sizing
4. Логіка адаптивного time-stop

## Чек-лист поширених Pine Script v6 проблем

| Категорія | Проблема | Серйозність |
|-----------|----------|-------------|
| Strategy | `calc_on_every_tick=true` з виявленням патернів | Висока |
| Arrays | Зростання масивів без лімітів розміру | Середня |
| Security | Кілька `request.security()` виклики | Середня |
| Labels | Створення без механізму видалення | Низька |
| Math | Ділення на `syminfo.mintick` без захисту | Висока |
| State | Неініціалізовані `var` змінні на першому барі | Середня |

## Команди діагностики

Запустіть ці діагностичні шаблони:
```pinescript
// Діагностичний overlay для додавання зверху:
diagnostic_mode = input.bool(false, "Режим діагностики")
if diagnostic_mode
    // Примусова видимість внутрішніх станів
    label.new(bar_index, high, str.tostring(state.phase), xloc=xloc.bar_index)
```

## Шаблони виправлень

### Виправлення 1: Захист меж масиву
```pinescript
// РАНІШЕ:
float val = array.get(myArray, idx)

// ПІСЛЯ:
float val = array.size(myArray) > idx ? array.get(myArray, idx) : na
```

### Виправлення 2: Консолідація security викликів
```pinescript
// РАНІШЕ:
float htClose = request.security(..., close)
float htMa = request.security(..., ta.ema(close, 50))

// ПІСЛЯ:
[htClose, htMa] = request.security(..., [close, ta.ema(close, 50)], lookahead=barmerge.lookahead_off)
```

### Виправлення 3: Запобігання repaint
```pinescript
// РАНІШЕ:
if not na(ph)
    self.structure.lastSwingHigh := ph

// ПІСЛЯ:
if barstate.isconfirmed and not na(ph)
    self.structure.lastSwingHigh := ph
```

## Структура звіту
1. **Критичні помилки**: Обов'язково виправити перед live торгівлею
2. **Високий пріоритет**: Ймовірно викликають неправильні сигнали
3. **Середній пріоритет**: Проблеми продуктивності/оптимізації
4. **Низький пріоритет**: Пропозиції щодо якості коду
5. **Позитивні відкриття**: Елементи, які працюють коректно
