# Агент діагностики торгівлі Pine Script v6

Цей агент забезпечує комплексну діагностику SMC-базованих стратегій Pine Script v6.

## Основні функції

### 1. Діагностика конечного автомата
```pinescript
var MarketState state = MarketState.new(
    AgentPhase.Perception,
    TrendState.Neutral, 
    AgentStatus.Idle,
    ...
)

// ТОЧКИ ДІАГНОСТИКИ:
// - фазові переходи на кожному барі
// - зміни статусу при зміні позиції
// - логіка таймера cooldown
```

### 2. Перевірка управління ризиком
```pinescript
// ПЕРЕВІРЯЙТЕ ЦІ РОЗРАХУНКИ:
// - Розмір позиції = (equity * risk_pct) / risk_per_share
// - Відстань SL = ATR * множник
// - Рівні TP: entry ± (risk * RR_multiple)
```

### 3. Цілісність Order Block
```pinescript
// КРИТЕРІЇ ВАЛІДАЦІЇ:
// - OB створюється після liquidity sweep
// - OB має FVG підтвердження
// - OB не в періоді cooldown
```

### 4. Виявлення liquidity sweep
```pinescript
// ЛОГІКА SWEEP ДЛЯ ПЕРЕВІРКИ:
bool sweptHighNow = high > highestLiq[1] and close < highestLiq[1]
bool sweptLowNow = low < lowestLiq[1] and close > lowestLiq[1]
```

## Виявлення критичних помилок

### Високий пріоритет
1. **NaN поширення**: Будь-який `na()` у розрахунках ризику викликає відхилення замовлень
2. **Security repaint**: `lookahead=barmerge.lookahead_on` викликає витік даних майбутнього
3. **Зростання масивів**: Необмежені масиви викликають сповільнення/crash скрипту
4. **Конфлікти замовлень**: Кілька `strategy.entry()` з однаковим ID

### Середній пріоритет
1. **Неспрямованість часових сесій**: UTC vs часовий пояс графика
2. **Ділення на нуль у context scoring**: Коли contextCount=0
3. **Розрахунок рівня POV**: Використання неправильного pivot таймфрейму
4. **Блокування кореляції**: Неправильний поріг кореляції
